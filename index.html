Endpoints
POST
/auth/accesstokenrequest
Request a new access token. Returns accessToken, mdAccessToken, userId.
GET
/auth/renewAccessToken
Extend token lifetime. Call ~15 min before expiry. Tokens last 90 minutes.
GET
/auth/me
Returns authenticated user profile. Use to verify a token is valid.
POST
/auth/oauthtoken
Exchange an OAuth authorization code for an access token.
Token request body
{ "name": "your_username", "password": "your_password", "appId": "Sample App", "appVersion": "1.0", "cid": 0, "sec": "your_api_secret" }
Authorization header
Authorization: Bearer <accessToken> Content-Type: application/json
Key rules
▸ Max 2 concurrent sessions — a 3rd login closes the oldest
▸ Renew ~15 min before expiry via /auth/renewAccessToken
▸ Use a dedicated API user — never share with browser logins
▸ Request once per app instance, distribute the token
Orders
POST
/order/placeOrder
Place a new order. Requires accountId, action (Buy/Sell), symbol, orderQty, orderType. Set isAutomated: true for algo orders.
POST
/order/modifyOrder
Modify an existing order by orderId.
POST
/order/cancelOrder
Cancel an order by orderId.
POST
/order/liquidatePosition
Liquidate an open position for a given account and contract.
GET
/order/list
List all orders for the authenticated user.

Connection pattern
const ws = new WebSocket('wss://demo.tradovateapi.com/v1/websocket'); ws.onopen = () => { ws.send(`authorize\n0\n\n${accessToken}`); }; ws.onmessage = (msg) => { if (msg.data === 'o' || msg.data === 'h') return; const data = JSON.parse(msg.data.slice(1)); };
Market data subscriptions require mdAccessToken, not accessToken.
Built-in snippets
Get access token (JS)
async function getAccessToken() { const res = await fetch('https://demo.tradovateapi.com/v1/auth/accesstokenrequest', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ name: process.env.TV_USERNAME, password: process.env.TV_PASSWORD, appId: 'My App', appVersion: '1.0', cid: 0, sec: process.env.TV_SECRET }) }); return res.json(); // { accessToken, mdAccessToken, userId } }
Place a market order (JS)
async function placeOrder(token, accountId) { const res = await fetch('https://demo.tradovateapi.com/v1/order/placeOrder', { method: 'POST', headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` }, body: JSON.stringify({ accountId, action: 'Buy', symbol: 'ESZ4', orderQty: 1, orderType: 'Market', isAutomated: true }) }); return res.json(); }

Built-in gotchas
Auth
2 concurrent session limit
A 3rd login closes the oldest session. Use one dedicated API user and distribute the token.
Market Data
mdAccessToken is separate from accessToken
Both returned by /auth/accesstokenrequest. Market data endpoints require mdAccessToken.
WebSocket
Messages are prefixed with a frame type character
Strip first char before JSON.parse() — msg.data.slice(1). a=data, h=heartbeat, o=open, c=close.
Orders
isAutomated flag required for algo orders
Set isAutomated: true in order body. Omitting it can cause rejections on live accounts.

User management is handled directly in your Supabase dashboard. Click below to open it.

Invite a user

Authentication → Users → Invite user → enter their email

Remove a user

Authentication → Users → click the user → Delete user
How invitations work
1. Go to Supabase → Authentication → Users → Invite user
2. Enter their email — they get a link to set their password
3. Once they set a password, they can sign in to the wiki
4. To remove access, delete their user from the same screen
Wiki URL
Access
Access is managed via Supabase Auth. Only users you invite can sign in. Admins can manage users directly from the User Management panel in the sidebar.
Deployment steps
1. Upload tradovate-wiki.html to your GitHub repo
2. Deploy the admin-users Edge Function to Supabase
3. Settings → Pages → main branch → Save
4. Invite teammates from the User Management panel
